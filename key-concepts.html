Core DNS Concepts

Understanding DNS requires more than knowing that it translates names into IP addresses. The concepts below explain the main parts of DNS and why each one matters.

1. Domain Name System (DNS)

DNS is the system that translates human-friendly domain names into machine-readable IP addresses so users can reach websites and services on the internet.

2. Domain Name

A domain name is the human-readable address used to access a website, such as example.com. Domain names are easier for people to remember than numeric IP addresses.

3. IP Address

An IP address is the numeric address computers use to locate and communicate with each other on a network. IPv4 addresses look like 192.0.2.1, while IPv6 addresses are longer, such as 2001:db8::1.

4. Recursive Resolver

A recursive resolver receives DNS questions from a user’s device and does the work of finding the answer. It may check cached results first, then query other DNS servers until it finds the correct IP address.

5. Root Servers

Root servers are the starting point of DNS resolution. They direct queries to the appropriate top-level domain servers, such as servers for .com, .org, or .net.

6. Authoritative Name Server

An authoritative name server stores the official DNS records for a domain. When a resolver needs the final answer for a domain, the authoritative server provides the correct record.

7. DNS Records

DNS records are entries in a DNS database that define how a domain name behaves, including where it points and what services it supports, such as websites, email, and verification records.

8. DNS Caching

DNS caching stores previous lookup results temporarily. Caching improves performance because the browser, operating system, or resolver can reuse a recent answer instead of repeating the full lookup process every time.

9. Time To Live (TTL)

Time To Live, or TTL, controls how long a DNS record can be cached before it needs to be refreshed. A shorter TTL allows DNS changes to update faster, while a longer TTL can improve performance by reducing repeated lookups.

10. DNSSEC

DNSSEC is a set of security extensions that helps verify DNS responses are authentic and have not been altered in transit.

11. DoH and DoT

DNS over HTTPS and DNS over TLS are privacy-focused protocols that encrypt DNS traffic between the client and resolver.
